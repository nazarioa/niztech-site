process

Before a single line of code hits my screen, we need a game plan. We can’t get to where we need to go if we don’t know where we are going. Once we have established a direction, I execute, test, refine and repeat until we hit our mark. Then, high-fives all round; this part is very important.

  1. Planning
    0
    The who, what, where, how, and why. I'll ask questions so that I fully understand your goals. This is where I try to become an extension of you. Getting a statement-of-work right make the subsequent steps much faster and more enjoyable.
  2. Develop
    1
    For me the fun part begins. I'll work diligently on your project. Updates are provided during my progress. You should see the monument being built just as much as the builder.
  3. Deliver
    2
    This not only includes a finished product, but depending on the project it can include README files or training. It however, does not always mean the end.
  4. Repeat
    3
    The world changes. Software is beautiful because it can be made to adapt. I take what is learned and apply it anew.
